# pyodide-sandbox-guard Specification

## Purpose

Defines the Python execution sandbox that blocks user code from importing Pyodide's JavaScript bridge modules (`js`, `pyodide_js`, `pyodide`), clears pre-existing module references from `sys.modules`, and ensures the guard is injected before every user code execution without interfering with standard library imports.

## ADDED Requirements

### Requirement: User Python code cannot import js proxy modules

The Python execution sandbox SHALL block all attempts to import `js`, `pyodide_js`, or `pyodide` modules (and their submodules). When a user's Python code attempts to import any of these modules, an `ImportError` SHALL be raised.

#### Scenario: Import js is blocked

- **WHEN** user Python code contains `import js`
- **THEN** execution SHALL raise `ImportError`
- **AND** the testcase verdict SHALL be `RE` (Runtime Error)

#### Scenario: Import pyodide_js is blocked

- **WHEN** user Python code contains `import pyodide_js`
- **THEN** execution SHALL raise `ImportError`

#### Scenario: From js import is blocked

- **WHEN** user Python code contains `from js import fetch`
- **THEN** execution SHALL raise `ImportError`

#### Scenario: Submodule import is blocked

- **WHEN** user Python code contains `import js.something` or `import pyodide_js._core`
- **THEN** execution SHALL raise `ImportError`

### Requirement: Pre-existing js module references are cleared

Before user code execution, the sandbox SHALL remove any pre-loaded references to `js`, `pyodide_js`, and `pyodide` modules from `sys.modules`. This prevents user code from accessing cached module references that bypass the import guard.

#### Scenario: Cached module reference is unavailable

- **WHEN** user Python code accesses `sys.modules['js']` after the sandbox guard is installed
- **THEN** the key SHALL NOT exist in `sys.modules`

### Requirement: Sandbox guard does not affect standard library imports

The import guard SHALL NOT interfere with Python standard library modules. Modules such as `sys`, `io`, `math`, `json`, `re`, `collections`, and `itertools` SHALL remain importable.

#### Scenario: Standard library modules import successfully

- **WHEN** user Python code contains `import math` or `import json`
- **THEN** the import SHALL succeed without errors
- **AND** the module SHALL be fully functional

### Requirement: Sandbox guard is injected before user code in every execution

The sandbox guard code SHALL be part of the wrapped Python code generated by `buildWrappedCode()`. It SHALL execute before the user's code and after the `sys.settrace` op-counter setup.

#### Scenario: Guard executes in correct order

- **WHEN** `buildWrappedCode()` generates the full Python script
- **THEN** the execution order SHALL be: (1) op-counter setup, (2) sandbox guard, (3) stdin/stdout redirect, (4) user code
